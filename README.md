# 局域网屏幕墙 (LAN Screen Wall)

一个干净、开源、可审计的局域网屏幕墙工具。一台主控机以网格形式实时展示局域网内多台终端的屏幕画面，适用于机房、教室、办公区监控等场景。

## 特性

- **自动发现，即开即用** — 主控端启动后自动广播，采集端启动后自动搜索并连接，无需手动输入 IP
- **智能网格布局** — ≤4 台自动按田字格(2列)排布，>4 台动态计算最优行列，终端离线后自动重排
- **毛玻璃界面** — 深蓝灰主题 + 半透明渐变标题条，瓦片实时显示主机名与帧率
- **GUI 采集端** — 小型配置界面，支持令牌/端口/帧率/画质/显示器/缩放宽度设置，配置自动保存
- **开机自启** — 采集端支持一键开启开机自启动（写入注册表 Run 键）
- **断线重连** — 连接中断后自动重新搜索主控端并重连
- **实时调参** — 右键瓦片可调帧率/画质，设置值会被记住，下次打开对话框默认显示当前值
- **安全透明** — 不注入进程、不读写文件、不释放子程序，全部源码可审计

## 架构

```
┌──────────────┐                    ┌──────────────┐
│  采集端 agent │  UDP 发现 → TCP 连接 │ 主控端 hub    │
│ (每台被监控机)│ ──────────────────▶│ (监控机, GUI) │
│  屏幕采集推流 │   JPEG 帧流         │  网格展示     │
└──────────────┘                    └──────────────┘
       └─ 自动搜索 ─┘   └─ 广播存在 ─┘
```

| 组件 | 文件 | 说明 |
|------|------|------|
| 主控端 | `hub.py` | TCP 服务 + UDP 广播 + PyQt5 网格界面 |
| 采集端 | `agent.py` | PyQt5 GUI + mss 屏幕采集 + JPEG 推流 |
| 通信协议 | `protocol.py` | 长度前缀帧 + 令牌握手 + UDP 发现协议 |

## 快速开始

### 方式一：直接使用 exe（推荐）

1. 下载 `ScreenWallHub.exe` 和 `ScreenWallAgent.exe`
2. 监控机运行 `ScreenWallHub.exe`
3. 每台被监控机运行 `ScreenWallAgent.exe`（自动搜索连接，无需配置）

### 方式二：从源码运行

需要 Python 3.8+，安装依赖：

```bash
pip install -r requirements.txt
```

**启动主控端（监控机）：**

```bash
python hub.py --port 5000 --token mySecret
```

**启动采集端（被监控机）：**

```bash
python agent.py
```

采集端启动后会弹出 GUI 界面，主机 IP 留空即自动发现，点"连接"或等待自动连接。

## 使用说明

### 主控端

启动后窗口标题显示本机 IP 和端口，采集端会自动发现并连接。

**网格布局规则：**

| 在线终端数 | 布局 |
|-----------|------|
| 1 台 | 1×1 |
| 2 台 | 1×2 |
| 3-4 台 | 2×2 田字格 |
| 5-6 台 | 2×3 |
| 7-9 台 | 3×3 |
| 10+ 台 | 按列数 = ceil(√n) 动态排布 |

终端离线后网格自动收缩重排，不会留下空位。

**右键瓦片操作：**

- 设置帧率（1-30 FPS，对话框默认显示当前值）
- 设置画质（JPEG 10-95，对话框默认显示当前值）
- 断开此终端

**命令行参数：**

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--port` | 5000 | TCP 监听端口 |
| `--token` | change-me | 连接令牌（**务必修改**） |
| `--cols` | 0 | 固定列数，0=自动排布 |

### 采集端

启动后显示 GUI 配置界面，**启动后自动连接**（无需手动点连接按钮）。

**配置项：**

| 配置 | 默认值 | 说明 |
|------|--------|------|
| 令牌 | change-me | 需与主控端一致 |
| 端口 | 5000 | 需与主控端一致 |
| 主机IP | （空） | 留空=自动发现主控端；也可手动填写 IP |
| 帧率 | 8 FPS | 屏幕采集帧率 |
| 画质 | 55 | JPEG 质量，越低越省带宽 |
| 最大宽度 | 1280 | 缩放宽度，0=不缩放 |
| 显示器 | 1 | 1=全部虚拟桌面，2=主屏，3+=副屏 |

**开机自启：** 勾选"开机自启动"复选框，采集端会写入注册表 `HKCU\...\Run` 键，开机后自动运行并连接。

**断线重连：** 连接中断后自动重新搜索主控端（通过 UDP 广播），找到后自动重连。

**配置持久化：** 所有配置自动保存到 `%APPDATA%\ScreenWallAgent\config.json`，下次启动自动加载。

## 安全设计

| 项目 | 做法 |
|------|------|
| 认证 | 连接必须携带令牌(Token)，主控端校验后才接受画面 |
| 权限 | 采集端**只发送**本机画面，不接受任何远程指令（仅接受帧率/画质参数） |
| 行为 | 无进程注入、无文件读写、无子进程释放 |
| 网络 | 仅局域网 TCP + UDP 广播，无外联 |
| 依赖 | 全部为知名开源库(PyQt5/mss/Pillow)，可审计 |

## 带宽估算

单台 1080p、质量 55、宽度缩放至 1280、8fps，实测约 **40 KB/帧 ≈ 320 KB/s**。10 台终端约 3 MB/s，百兆局域网轻松承载。降低画质或帧率可进一步省带宽。

## 从源码打包 exe

```bash
pip install pyinstaller

# 主控端
pyinstaller --onefile --windowed --name ScreenWallHub \
  --icon hub.ico --add-data "hub.ico;." --add-data "hub.png;." \
  --collect-all PyQt5 --collect-all mss hub.py

# 采集端
pyinstaller --onefile --windowed --name ScreenWallAgent \
  --icon agent.ico --add-data "agent.ico;." --add-data "agent.png;." \
  --collect-all PyQt5 --collect-all mss agent.py
```

打包后的 exe 在 `dist/` 目录下。

## 常见问题

**Q: 采集端连不上主控端？**
- 检查两端的令牌和端口是否一致
- 检查主控端防火墙是否放行了对应端口（默认 5000 TCP + 5001 UDP）
- 采集端主机 IP 留空走自动发现；如自动发现失败，可手动填写主控端 IP

**Q: 防火墙拦截？**
- 主控端需放行 TCP 端口（默认 5000）入站 + UDP 端口（默认 5001）入站
- 采集端为出站连接，一般无需配置

**Q: 多显示器只想要副屏？**
- 采集端 GUI 显示器编号选 2（主屏）或 3+（副屏）。编号对应 mss 的 monitors 顺序

**Q: 运行时图标偏小？**
- 已通过 PNG 多尺寸 addPixmap 修复。如仍偏小，清除 Windows 图标缓存：`ie4uinit.exe -show`

**Q: exe 首次启动慢？**
- PyInstaller 单文件模式首次运行需解包，几秒后正常，属预期行为
